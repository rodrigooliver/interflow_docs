### Automatic File Cleanup

Automatically manage your organization's storage with the new automatic file cleanup feature.

#### What is it for?

- Automatically free up storage space when reaching a configured limit
- Intelligently delete old files while keeping the most recent ones
- Protect important files linked to Prompts, Shortcuts, Flows, Integrations, and Deployments

#### How to use?

1. In the left sidebar, click on **Settings**
2. Go to the **Usage** tab
3. Click on the **Storage** card
4. In the panel that opens, scroll to the **Auto Cleanup** section
5. Turn on the toggle switch
6. Click the gear icon ‚öôÔ∏è to adjust options:
   - **Minimum age**: files newer than this period will not be deleted
   - **Usage trigger**: cleanup only starts when storage exceeds this limit
   - **Don't delete below**: cleanup stops when usage reaches this level
   - **Maximum per run**: maximum number of files deleted at a time
   - **Execution time**: time of day when cleanup runs daily
7. Click **Save**

::: warning ‚ö†Ô∏è Important
Files linked to **AI Prompts**, **Message Shortcuts**, **Flows**, **Integrations**, and **Deployments** are never automatically deleted.
:::

#### Configurable parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| Minimum age | Newer files will not be deleted | 30 days |
| Usage trigger | Usage percentage that triggers cleanup | 80% |
| Don't delete below | Minimum usage percentage after cleanup | 70% |
| Maximum per run | File limit per cycle | 100 |
| Execution time | Daily cleanup time | 03:00 |
