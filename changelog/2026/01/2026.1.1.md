## Novas Funcionalidades

### Templates de WhatsApp Oficial em Disparo em Massa

Agora é possível enviar **templates de WhatsApp Oficial** através do sistema de **disparo em massa**! Esta funcionalidade permite que você utilize templates pré-aprovados pelo WhatsApp para comunicações em larga escala, garantindo maior taxa de entrega e conformidade com as políticas do WhatsApp Business API.

#### Para que serve?

- Enviar **mensagens padronizadas e aprovadas** para múltiplos contatos
- Utilizar **templates com variáveis dinâmicas** para personalização
- Garantir **alta taxa de entrega** através da API oficial do WhatsApp
- Manter **conformidade** com as políticas do WhatsApp Business
- Enviar mensagens para contatos que **não iniciaram conversa** (janela de 24h não se aplica)

#### Requisitos

::: warning Importante
Para utilizar esta funcionalidade, é necessário:
- ✅ Ter um canal **WhatsApp Oficial** configurado
- ✅ **Criar e aprovar** templates no Meta Business Manager
- ✅ **Sincronizar** os templates aprovados no sistema Interflow
:::

#### Como usar?

1. **Crie seus templates** no Meta Business Manager
2. Aguarde a **aprovação** dos templates pelo WhatsApp
3. No Interflow, acesse **Canais → WhatsApp Oficial**
4. Clique em **"Sincronizar Templates"** para importar os templates aprovados
5. Acesse **Disparo em Massa** no menu
6. **Selecione o tipo de mensagem**: "Template WhatsApp Oficial"
7. **Escolha o template** desejado da lista
8. **Preencha as variáveis** do template (se houver)
9. **Selecione os destinatários** (contatos, grupos, segmentos)
10. **Agende ou envie imediatamente**

#### Recursos

- ✅ Suporte a **variáveis dinâmicas** nos templates
- ✅ **Pré-visualização** do template antes do envio
- ✅ **Agendamento** de envios
- ✅ **Relatórios detalhados** de entrega e leitura
- ✅ **Validação automática** de templates sincronizados

### Fluxos Acionados em Qualquer Mensagem do Cliente

O sistema de **fluxos automáticos** agora pode ser **acionado em qualquer mensagem enviada pelo cliente**, não apenas na primeira mensagem da conversa! Esta funcionalidade oferece muito mais flexibilidade na automação de atendimentos e permite criar experiências conversacionais mais inteligentes.

#### Para que serve?

- Acionar **automações** a qualquer momento da conversa
- Responder automaticamente a **palavras-chave específicas**
- Criar **menus interativos** durante o atendimento
- Implementar **comandos** que o cliente pode usar a qualquer momento
- **Segmentar conversas** baseado em intenções do cliente

#### Como funciona?

**Anteriormente**: Fluxos só eram acionados na primeira mensagem do cliente (início da conversa)

**Agora**: Você pode configurar fluxos que monitoram **todas as mensagens** do cliente e são acionados quando condições específicas são atendidas!

#### Como configurar?

1. Acesse **Fluxos** no menu
2. **Crie ou edite** um fluxo
3. Nas **Configurações do Fluxo**, defina:
   - **Quando acionar**: "Em qualquer mensagem" (nova opção!)
   - **Condições**: Configure as regras de acionamento
4. **Salve** o fluxo

#### Nova Funcionalidade: Condições de Verificação na Última Mensagem

Dentro dos fluxos, agora você pode adicionar **nós de condição** que verificam a última mensagem enviada pelo cliente!

##### Para que serve?

- **Identificar palavras-chave** na mensagem do cliente
- **Direcionar o fluxo** baseado no conteúdo da mensagem
- Criar **menus dinâmicos** que respondem ao input do cliente
- Implementar **validações** de respostas
- **Classificar intenções** automaticamente

##### Como usar?

1. No **editor de fluxos**, adicione um nó **"Condição"**
2. Selecione o tipo: **"Verificar última mensagem do cliente"**
3. Configure as regras:
   - **Contém**: Verifica se a mensagem contém uma palavra/frase
   - **É igual a**: Verifica se a mensagem é exatamente igual
   - **Começa com**: Verifica se a mensagem inicia com algo
   - **Termina com**: Verifica se a mensagem termina com algo
   - **Expressão regular**: Para validações avançadas
4. Defina **múltiplas palavras-chave** (ex: "sim", "yes", "confirmar")
5. Configure as **ações** para cada resultado (verdadeiro/falso)
6. **Conecte** aos próximos nós do fluxo

##### Exemplos de Uso

**Exemplo 1: Menu de Opções**
```
Cliente envia: "Quero suporte"
→ Fluxo é acionado
→ Sistema responde: "Digite 1 para vendas, 2 para suporte técnico"
→ Condição verifica se resposta contém "1" ou "2"
→ Direciona para o departamento correto
```

**Exemplo 2: Validação de Confirmação**
```
Sistema pergunta: "Confirma o agendamento para amanhã às 14h?"
→ Condição verifica se mensagem contém: "sim", "confirmo", "ok", "yes"
→ Se SIM: Confirma agendamento
→ Se NÃO: Solicita nova data
```

**Exemplo 3: Palavras-chave de Atendimento**
```
Cliente em atendimento envia: "cancelar pedido"
→ Fluxo específico é acionado automaticamente
→ Sistema inicia processo de cancelamento
→ Transfere para equipe de cancelamentos
```

**Exemplo 4: Comandos Rápidos**
```
Cliente envia: "/status"
→ Fluxo detecta comando
→ Retorna status do pedido automaticamente
```

#### Recursos

- ✅ **Múltiplos gatilhos**: Configure várias palavras-chave por condição
- ✅ **Case insensitive**: Não diferencia maiúsculas/minúsculas
- ✅ **Suporte a regex**: Validações avançadas com expressões regulares
- ✅ **Priorização**: Defina ordem de execução dos fluxos
- ✅ **Teste em tempo real**: Teste suas condições no editor de fluxos
- ✅ **Logs detalhados**: Acompanhe quais fluxos foram acionados e por quê
