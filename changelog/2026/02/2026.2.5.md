## ✨ Novas Funcionalidades

### Integração de Fallback para Agentes IA

Configure uma integração OpenAI de reserva para que seus Agentes IA continuem funcionando automaticamente, mesmo quando os créditos da Interflow acabarem. Garanta continuidade no atendimento sem interrupções!

#### Para que serve?

- **Continuidade garantida**: Seus agentes IA nunca param de funcionar por falta de créditos
- **Transição automática**: Quando os créditos Interflow acabam, o sistema usa sua API OpenAI automaticamente
- **Controle de custos**: Você decide quando usar créditos Interflow ou sua própria conta OpenAI
- **Sem perda de atendimentos**: Conversas importantes não são interrompidas por falta de saldo

#### Como usar?

##### 1. Criar Integração OpenAI (se ainda não tiver)

1. No **menu lateral esquerdo**, clique em **"Configurações"** ⚙️
2. Clique na aba **"Integrações"**
3. Localize a seção **"OpenAI"** e clique em **"Adicionar"**
4. Preencha:
   - **Nome**: Ex: "OpenAI - Fallback Principal"
   - **API Key**: Cole sua chave da OpenAI
     - Obtenha em: https://platform.openai.com/api-keys
5. Clique em **"Salvar"**

##### 2. Configurar Fallback no Agente IA

1. No **menu lateral esquerdo**, clique em **"Agentes IA"** 🤖
2. Clique em **"Editar"** no agente desejado
3. Vá para a aba **"Avançado"**
4. Na seção **"Integração de Fallback"** (fundo azul), selecione sua integração OpenAI
5. Clique em **"Salvar"**

#### Como funciona?

**Cenário Normal (Com Créditos):**
```
Cliente envia mensagem
    ↓
Agente verifica créditos Interflow
    ↓
✅ Tem créditos disponíveis
    ↓
Usa Interflow (incluído no plano)
    ↓
Responde ao cliente
```

**Cenário Fallback (Sem Créditos):**
```
Cliente envia mensagem
    ↓
Agente verifica créditos Interflow
    ↓
❌ Créditos esgotados
    ↓
🔄 Ativa integração de fallback
    ↓
Usa sua API OpenAI
    ↓
Responde ao cliente normalmente
```

## 🔧 Melhorias

### Interface de Configuração

- **Visual destacado**: Seção de fallback com fundo azul para fácil identificação
- **Instruções claras**: Descrição explicando quando o fallback será usado
- **Seleção intuitiva**: Lista todas as integrações OpenAI disponíveis
- **Opção de remover**: Possibilidade de não configurar fallback

### Sistema de Alertas

- **Notificação de ativação**: Você é alertado quando o fallback é usado
- **Logs detalhados**: Registro completo de quando cada integração foi utilizada
- **Monitoramento**: Acompanhe qual fonte de créditos está sendo usada

### Performance

- **Consulta otimizada**: Fallback só é consultado quando necessário (créditos esgotados)
- **Transição instantânea**: Troca automática sem delay perceptível
- **Zero overhead**: Não adiciona latência nas execuções normais

## 🎯 Benefícios

- ✅ **Sem interrupções**: Atendimento contínuo mesmo sem créditos Interflow
- ✅ **Automático**: Transição transparente, sem necessidade de intervenção manual
- ✅ **Flexível**: Configure fallback apenas nos agentes críticos
- ✅ **Seguro**: Chaves API criptografadas e protegidas
- ✅ **Econômico**: Use primeiro os créditos Interflow, depois sua API
- ✅ **Visibilidade**: Saiba exatamente quando e qual integração foi usada

---

## 🔒 Segurança

- **Criptografia**: Todas as chaves API são criptografadas no banco de dados
- **Privacidade**: Suas chaves nunca são expostas na interface
- **Controle**: Apenas você tem acesso às suas integrações
- **Revogação**: Você pode revogar chaves a qualquer momento na OpenAI

---

## 📊 Casos de Uso

### 1. Atendimento 24/7 Crítico
Configure fallback em agentes de suporte técnico urgente para garantir que sempre haverá resposta, independente do horário ou saldo disponível.

### 2. Picos de Demanda
Durante eventos especiais ou promoções, quando o volume de atendimentos aumenta drasticamente, o fallback garante capacidade adicional.

### 3. Período de Transição
Ao migrar de créditos Interflow para uso de sua própria API, configure o fallback para uma transição suave.

---

::: tip 💡 Dica
Configure o fallback **preventivamente**, não espere os créditos acabarem! Assim você garante que nunca haverá interrupção no atendimento, mesmo em situações inesperadas.
:::

::: warning ⚠️ Atenção
O fallback **só é usado quando os créditos Interflow acabam**. Durante o uso normal, você continua consumindo os créditos incluídos no seu plano, que geralmente têm melhor custo-benefício.
:::
