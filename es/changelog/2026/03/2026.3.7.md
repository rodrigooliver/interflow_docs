## ✨ Nuevas Funcionalidades

### Reenviar mensajes a una nueva conversación

Puedes **reenviar** un mensaje del chat (texto o medio con URL pública HTTPS) a un **nuevo contacto o canal**, reutilizando el flujo de iniciar conversación.

#### ¿Para qué sirve?

- **Agilidad** – Reenvía contenido a otro cliente o canal sin copiar/pegar a mano
- **Medios por URL** – Imágenes, vídeos, audios y documentos se envían cuando el adjunto tiene una URL HTTPS válida

#### ¿Cómo usarlo?

1. En el chat, abre el menú del mensaje (tres puntos)
2. Elige **Reenviar**
3. Selecciona el contacto y el canal
4. Al finalizar, un **toast** ofrece el botón **Ir a la conversación** — permaneces en el chat actual hasta que decidas abrir la nueva atención

---

### Mensaje inicial tipado en la API `POST /chat/create`

El cuerpo acepta `initialMessage` como **cadena** (texto simple, como antes) o como **objeto** con tipo y, para medios, URL HTTPS.

#### Formato texto

```json
{
  "type": "text",
  "content": "Tu mensaje aquí"
}
```

#### Formato medio (p. ej. reenvío)

```json
{
  "type": "image",
  "content": "Leyenda opcional",
  "url": "https://...",
  "name": "archivo.jpg",
  "mimetype": "image/jpeg"
}
```

Tipos admitidos: `text`, `image`, `video`, `audio`, `document`. La documentación externa de la API incluye ejemplos.

---

### Atención automática al crear con mensaje inicial

Cuando hay **mensaje inicial** que enviar y el usuario está **autenticado** como agente:

- Chat **pendiente** → efecto equivalente a **Atender**: estado `in_progress`, responsable actual, mensaje de sistema y reglas de pausa de flujo cuando aplique (clientes spam no disparan atención automática)
- Chat ya **en curso** con otro responsable → si no eres colaborador, se te **añade como colaborador** (sin duplicar si ya participas)

Si **no** hay mensaje inicial, la creación del chat se comporta como antes.

## 🎯 Beneficios

- ✅ **Reenvío integrado al flujo de nueva conversación** – Menos clics y menos errores operativos
- ✅ **API flexible** – Texto o medio por URL en la misma ruta de creación de chat
- ✅ **Contexto de atención alineado** – Nuevo chat ya en curso cuando corresponde, o entras como colaborador
- ✅ **UX en el reenvío** – Tú decides cuándo abrir la nueva conversación (toast con acción)
